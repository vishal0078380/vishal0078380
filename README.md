### **1. Secure Static Website Hosting on AWS**  
**Tech Stack:** `S3` `CloudFront` `Route 53` `AWS WAF` `CloudWatch` `ACM`  

**Architecture Highlights:**
- **Security-First Design:** Implemented S3 bucket policies blocking public access, encrypted objects with SSE-S3
- **Global Delivery:** Configured CloudFront distribution with geo-restrictions and OAI for secure S3 access
- **Monitoring:** Established CloudWatch alarms for 4xx/5xx errors and request volume anomalies
- **Outcome:** 99.9% availability with DDoS protection through AWS Shield Standard

### **2. IAM & Access Control Laboratory**  
**Tech Stack:** `IAM` `AWS Organizations` `Service Control Policies` `IAM Roles`  

**Security Implementation:**
- **Least Privilege Model:** Created granular IAM policies with conditions and explicit denies
- **Role Separation:** Implemented cross-account roles for secure AWS Organization management
- **Policy Validation:** Tested policies using IAM simulator before production deployment
- **Outcome:** Zero standing privileges architecture with all access requiring assumption of roles

### **3. Three-Tier Web Application Architecture**  
**Tech Stack:** `VPC` `Public/Private Subnets` `NAT Gateway` `Application Load Balancer` `Auto Scaling`  

**Infrastructure Design:**
- **Network Isolation:** Deployed multi-AZ VPC with public, private, and database subnets
- **Security Layering:** Implemented security groups with stateful filtering and NACLs for subnet-level control
- **Scalability:** Configured Auto Scaling groups with health checks and scaling policies
- **Outcome:** Fault-tolerant architecture surviving AZ failures with automated recovery

### **4. Backend API Deployment on EC2**  
**Tech Stack:** `EC2` `Docker` `Nginx` `Node.js` `PM2` `Security Groups`  

<div align="center">
  <img src="https://media.giphy.com/media/ZVik7pBtu9dNS/giphy.gif" width="300" alt="API Deployment" />
</div>

**Deployment Architecture:**
- **Reverse Proxy Setup:** Nginx configuration for SSL termination, compression, and request routing
- **Containerization:** Dockerized Node.js application with multi-stage builds for optimization
- **Process Management:** Implemented PM2 for process management and zero-downtime reloads
- **Outcome:** Production-ready API deployment with monitoring, logging, and health checks

### **5. Backend with Private RDS Database**  
**Tech Stack:** `RDS PostgreSQL` `Private Subnets` `VPC Endpoints` `Parameter Store`  

**Database Security Pattern:**
- **Network Isolation:** RDS instance deployed in private subnets with no public IP
- **Connection Security:** Implemented SSL encryption for application-to-database traffic
- **Secret Management:** Used Systems Manager Parameter Store for database credentials
- **Outcome:** Compliant database architecture meeting data isolation requirements
